As to my past experience on machine learning and system, there are mainly three parts: system, machine learning for system, and system for machine learning.

My work in Baidu focuses on promoting the performance and efficiency of web search engine with methods of machine learning, including ANN on document matching and NLP models on ranking, and we also build system for machine learning services focusing on deploying on XPU devices including CPU, GPU, and Kunlun for online predicting.

My undergraduate thesis applys reinforcement learning on traffic scheduling in Remote Procedure Call framework, inproving the average latency by predicting long latency queries. The idea came from my experience when I was tracing the long latency queries during my internship in Bytedance. I niticed a searching query would go down from query analysis, recalling documents, and ranking, with each service having different computational complexity, leading to different latency for the same query. On the other hand, in the same service, the calculation would have different time cost due to machine status that the query is routed to, for example a machine with heavy CPU load will do poor at feature extracting and other operations implemented by CPU. Therefore, I logged the latency down into message queue and collected billions of possible features that were possibly lead to long latency, and tried to find the relavance for the prediction of long latency queries.

During the one-and-a-half years’ work, I was thrilled when I successfully abstracted the repetitive code in a systematic way on the Click-Through-Rate project.
